i-24 motion project

dates worked: august 2022 - present

In the fall of 2022, I worked as part of the i-24 motion project, whose project link can be found here. Within this position, I did quantitative data analysis of car trajectory data given from the hundreds of cameras installed over interstate 24, which runs through Nashville. These cameras capture millions of data points, each locating the position of each car within frame of the cameras. In the fall, I developed code to accomplish 5 main goals:

  1. calculate data on dynamics — speed and acceleration — on cars along with interstate, and generate basic statistics on these
  2. calculate data on lane occupation and changes — namely, where cars are occupied on the interstate and whether or not cars change lanes
  3. analyze conditional probability of lane change given acceleration and vice versa, to see the correlational relationship between the two variables. this is important for later analysis and predictions on when we might expect a lane change or an acceleration to aid with driver simulation.
  4. calculate the leader (car immediately in front) of a car as well as changes in leaders.

In spring of 2022, I will worked on producing scaleable, efficient code that will work on larger data sets at once to display calculated data on a centralized dashboard to show the lab's work to internal and external clients. To do this, I:

  1. utilized multi-threading to optimize the amount of time spent processing data by half!
  2. process data based on the above five goals, as well as trajectory lengths, vehicle types, etc. on hundreds of thousands of trajectories per document
  3. created a dashboard with animated plots from calculated data, where each snapshot is made from a sliding window approach to data processing

skills used: python, github
